AWS provideed security and compliance, also provide compliance controls that help customers meet requirements fro regulatory agencies.

Security and compliance services
- AWS Security Hub - provide security posture and view of security and compliance status across AWS environment.
- AWS Aritfact - self service protal where user can access compliance report.
- Amazon Inspector - will continue scanning for vulnerabilites and check if AWS resources is according to the need.
- Amazon GuardDuty - a managed threat detection service, uses threat intelligence and Machine learning
- AWS Shield - always-on detection to detect, remediate, and prevent DoS and DDoS.

AWS IAM (Identity & Access Management) overview
- Enables access to AWS resources securely, manages access of who can interacts with AWS
- Each IAM guser, group, or role, will inherits permission of the group
- Each user will have individual security credential
- By default, access to file will be denied unless said otherwise
- Offered as feature of AWS account, with no additional charges
- Can control time of the day login is allowed, source IP address, MFA

Attach IAM policy to IAM entities which consist of statements,
- version - specify the version of the policy language
- SID (tpional) - specify statement ID
- Effect - to allow or to deny, but default is deny
- Principal - the IAM entities that will be effected
- Action - list of action to allow or deny
- Resources - mention the specific resource the action will affect
- Conditions - the requirements, can be either by time, login MFA, etc

AWS Identity and Access Management - IAM Access analyzer
- Analyze the resources that will be available to outside an AWS account based on the resource policy
- Analyze permissions using Amazon S3 buckets, AWS KMS keys, Amazon SQS queus, IAM roles, AWS Lambda functions
